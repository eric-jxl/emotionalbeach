## 🎯 项目组织结构详解

### 业务逻辑层
```
项目入口 (main.go)
    ↓
路由配置 (router.go)
    ↓
┌─────────────────────────────────┐
│  Controller 层 (HTTP处理)        │
├─────────────────────────────────┤
│ • user.go (用户接口)             │
│ • github.go (OAuth)             │
│ • relation.go (好友管理)        │
│ • preload_cache.go (缓存预热)   │
└─────────────────────────────────┘
    ↓
┌─────────────────────────────────┐
│  Service 层 (业务逻辑)          │
├─────────────────────────────────┤
│ • webhook.go (邮件发送)         │
└─────────────────────────────────┘
    ↓
┌─────────────────────────────────┐
│  DAO 层 (数据访问)              │
├─────────────────────────────────┤
│ • user.go (用户表操作)          │
│ • relation.go (关系表操作)      │
└─────────────────────────────────┘
    ↓
📊 数据库 (PostgreSQL/MySQL)
🔴 Redis 缓存
```

### 中间件栈
```
HTTP 请求
    ↓
Recovery 中间件 (错误恢复)
    ↓
Logger 中间件 (请求日志)
    ↓
RateLimit 中间件 (IP限流)
    ↓
Assets Cache 中间件 (静态资源缓存)
    ↓
JWT 中间件 (认证验证)
    ↓
CORS 中间件 (跨域处理)
    ↓
业务处理
```
